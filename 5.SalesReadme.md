# 🚀 Full-Stack Next.js 프로젝트 구축 가이드 (Clerk & MongoDB)

이 문서는 Next.js를 기반으로 Clerk 인증 시스템과 MongoDB 데이터베이스를 연동하는 풀스택 애플리케이션 구축 단계를 설명합니다.

## 1. 프로젝트 초기 설정

먼저 Next.js 프로젝트를 생성하고 필요한 패키지를 설치합니다.

```bash
# 1. Next.js 프로젝트 생성
npx create-next-app@latest my-fullstack-app
# 설정: TypeScript: Yes, ESLint: Yes, Tailwind CSS: Yes, src/ directory: Yes, App Router: Yes, Import alias: Yes (@/*)

cd my-fullstack-app

# 2. 필수 라이브러리 설치
npm install @clerk/nextjs mongoose svix
```

## 2. Clerk 인증 설정 (Authentication)

1. [Clerk Dashboard](https://dashboard.clerk.com/)에서 새 프로젝트를 생성합니다.
2. `.env.local` 파일을 루트 폴더에 만들고 API 키를 복사합니다.

```env
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...
CLERK_SECRET_KEY=sk_test_...

NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up
NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL=/
NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL=/
```

3. `src/middleware.ts` 파일을 생성하여 보호할 경로를 설정합니다.

```typescript
import { authMiddleware } from "@clerk/nextjs";

export default authMiddleware({
  // 인증이 필요 없는 공개 경로 설정 (예: 웹훅 엔드포인트)
  publicRoutes: ["/api/webhooks/clerk"],
});

export const config = {
  matcher: ["/((?!.+\\.[\\w]+$|_next).*)", "/", "/(api|trpc)(.*)"],
};
```

4. `src/app/layout.tsx`를 `<ClerkProvider>`로 감쌉니다.

```typescript
import { ClerkProvider } from '@clerk/nextjs'

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <ClerkProvider>
      <html lang="ko">
        <body>{children}</body>
      </html>
    </ClerkProvider>
  )
}
```

## 3. MongoDB 연결 설정

1. [MongoDB Atlas](https://www.mongodb.com/cloud/atlas)에서 클러스터를 생성하고 연결 문자열(URI)을 가져옵니다.
2. `.env.local`에 추가합니다.
   `MONGODB_URL=mongodb+srv://<username>:<password>@cluster...`

3. `src/lib/db.ts` (또는 `mongoose.ts`) 파일을 생성하여 연결 로직을 작성합니다.

```typescript
import mongoose from 'mongoose';

const MONGODB_URL = process.env.MONGODB_URL;

let cached = (global as any).mongoose || { conn: null, promise: null };

export const connectToDatabase = async () => {
  if (cached.conn) return cached.conn;

  if (!MONGODB_URL) throw new Error('MONGODB_URL is missing');

  cached.promise = cached.promise || mongoose.connect(MONGODB_URL, {
    dbName: 'my-database-name',
    bufferCommands: false,
  });

  cached.conn = await cached.promise;
  return cached.conn;
};
```

## 4. 데이터 모델 정의 (User 모델)

Clerk 유저 정보를 저장할 유저 스키마를 정의합니다.
`src/lib/models/user.model.ts`

```typescript
import { Schema, model, models } from "mongoose";

const UserSchema = new Schema({
  clerkId: { type: String, required: true, unique: true },
  email: { type: String, required: true, unique: true },
  username: { type: String, required: true },
  photo: { type: String, required: true },
  firstName: { type: String },
  lastName: { type: String },
});

const User = models?.User || model("User", UserSchema);
export default User;
```

## 5. Webhook을 통한 데이터 동기화 (Clerk ↔ MongoDB)

Clerk에서 회원가입이 일어나면 자동으로 우리 DB에도 저장되도록 웹훅을 설정합니다.

1. **Clerk Dashboard > Webhooks**에서 Endpoint를 추가합니다. (URL: `https://your-domain.com/api/webhooks/clerk`)
2. `WEBHOOK_SECRET`을 받아 `.env.local`에 추가합니다.
3. `src/app/api/webhooks/clerk/route.ts` 파일을 생성합니다.

```typescript
import { Webhook } from 'svix'
import { headers } from 'next/headers'
import { WebhookEvent } from '@clerk/nextjs/server'
import { createUser } from '@/lib/actions/user.actions' // 아래에서 구현

export async function POST(req: Request) {
  const WEBHOOK_SECRET = process.env.WEBHOOK_SECRET
  if (!WEBHOOK_SECRET) throw new Error('Secret missing')

  const headerPayload = headers();
  const svix_id = headerPayload.get("svix-id");
  const svix_timestamp = headerPayload.get("svix-timestamp");
  const svix_signature = headerPayload.get("svix-signature");

  const payload = await req.json()
  const body = JSON.stringify(payload);
  const wh = new Webhook(WEBHOOK_SECRET);

  let evt: WebhookEvent
  try {
    evt = wh.verify(body, {
      "svix-id": svix_id!,
      "svix-timestamp": svix_timestamp!,
      "svix-signature": svix_signature!,
    }) as WebhookEvent
  } catch (err) {
    return new Response('Error occured', { status: 400 })
  }

  const eventType = evt.type;

  if (eventType === 'user.created') {
    const { id, email_addresses, image_url, first_name, last_name, username } = evt.data;

    const user = {
      clerkId: id,
      email: email_addresses[0].email_address,
      username: username!,
      firstName: first_name,
      lastName: last_name,
      photo: image_url,
    }

    const newUser = await createUser(user); // DB 저장 함수 호출
    return Response.json({ message: 'OK', user: newUser })
  }

  return new Response('', { status: 200 })
}
```

## 6. Server Actions (DB 조작 함수)

`src/lib/actions/user.actions.ts` 파일을 만들어 실제 DB 처리를 담당하게 합니다.

```typescript
"use server";

import { connectToDatabase } from "../db";
import User from "../models/user.model";

export async function createUser(user: any) {
  try {
    await connectToDatabase();
    const newUser = await User.create(user);
    return JSON.parse(JSON.stringify(newUser));
  } catch (error) {
    console.error(error);
  }
}
```

## 8. 로컬 테스트 팁 (Webhook)

웹훅은 외부에서 내 서버로 접속해야 하므로 로컬 개발 시에는 **ngrok**이나 **Localtunnel**을 사용해야 합니다.

1. `npx borderless-local-webhook` 또는 `ngrok http 3000` 실행.
2. 생성된 URL을 Clerk Webhook URL에 입력.
3. 회원가입 테스트 진행 -> MongoDB Dashboard에서 데이터 생성 확인.

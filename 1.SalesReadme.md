# 🚀 Django 프로젝트 초기 설정 가이드

이 문서는 제공된 영상을 바탕으로 한 **Django 및 Django Rest Framework(DRF)** 초기 환경 설정 매뉴얼입니다. 판매 관리 시스템(`sales` 앱) 구축을 위한 기초 뼈대를 잡는 과정을 다룹니다.

## 1. 개발 환경 준비

먼저 프로젝트를 진행할 디렉토리를 생성하고 가상환경을 활성화합니다. (가상환경은 선택사항이나 권장됩니다.)

```bash
# 1. 프로젝트 폴더 생성 및 이동
mkdir django_sales_project
cd django_sales_project

# 2. 가상환경 생성 및 활성화 (Mac/Linux)
python3 -m venv .venv
source .venv/bin/activate

# 2. 가상환경 활성화 (Windows)
# .venv\Scripts\activate
```

## 2. Django 설치 및 프로젝트 초기화

영상에서 핵심적으로 다룬 깔끔한 폴더 구조를 만드는 방식입니다.

```bash
# 1. Django 설치
pip install django

# 2. 프로젝트 생성 (현재 디렉토리를 기준으로 'config' 설정 폴더 생성)
django-admin startproject config .

# 3. 기능 앱(sales) 생성
python manage.py startapp sales
```

## 3. 필수 관리 파일 및 데이터 스크립트 생성

터미널이나 탐색기를 통해 프로젝트 루트(`manage.py`가 있는 위치)에 다음 파일들을 생성합니다.

```bash
# 빈 파일 생성 (터미널 기준)
touch requirements.txt
touch readme.md
touch excel_data.py
touch model_data.py
```

## 4. 의존성 패키지 설정 (`requirements.txt`)

`requirements.txt` 파일을 열고 영상에서 명시한 라이브러리 버전을 입력합니다.

```text
Django>=4.2, <5.0
djangorestframework>=3.14
```

입력 후 아래 명령어로 설치를 완료합니다.
```bash
pip install -r requirements.txt
```

## 5. 프로젝트 설정 반영 (`config/settings.py`)

생성한 `sales` 앱과 `rest_framework`를 Django 프로젝트에 등록합니다. `config/settings.py` 파일을 열어 `INSTALLED_APPS` 항목을 수정하세요.

```python
# config/settings.py

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    
    # 3rd Party Apps
    'rest_framework',
    
    # Local Apps
    'sales',
]
```

## 6. 최종 폴더 구조 확인

설정이 완료되면 다음과 같은 구조를 갖게 됩니다.

```text
django_sales_project/
├── config/             # 프로젝트 설정 (settings, urls 등)
├── sales/              # 판매 관리 기능 앱
├── .venv/              # 파이썬 가상환경
├── manage.py           # Django 실행 스크립트
├── requirements.txt    # 설치 패키지 목록
├── readme.md           # 프로젝트 설명 파일
├── excel_data.py       # 엑셀 데이터 처리용 스크립트
└── model_data.py       # 모델 데이터 처리용 스크립트
```

## 7. 서버 실행 확인

모든 설정이 완료되었다면 서버를 실행하여 정상 작동 여부를 확인합니다.

```bash
python manage.py runserver
```
브라우저에서 `http://127.0.0.1:8000/`에 접속했을 때 Django 로켓 로고가 보이면 성공입니다.

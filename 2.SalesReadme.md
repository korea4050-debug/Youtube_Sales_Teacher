# 🚀 Django & DRF 프로젝트 초기 환경 설정 가이드

이 문서는 Django 프레임워크와 Django Rest Framework(DRF)를 사용하여 판매 관리(Sales) 및 데이터 처리 프로젝트를 구축하기 위한 초기 설정 단계를 설명합니다.

---

## 1. 프로젝트 폴더 및 가상환경 생성
터미널(또는 CMD)을 열고 프로젝트를 진행할 디렉토리를 생성합니다.

```bash
# 1. 프로젝트 폴더 생성 및 이동
mkdir django_sales_project
cd django_sales_project

# 2. 가상환경 생성 (파이썬 버전 독립성 유지)
python -m venv .venv

# 3. 가상환경 활성화
# Mac/Linux:
source .venv/bin/activate
# Windows:
# .venv\Scripts\activate
```

---

## 2. 프로젝트 및 앱 구조 생성
Django를 설치하고 프로젝트 메인 설정 폴더(`config`)와 기능 앱(`sales`)을 생성합니다.

```bash
# 1. Django 설치
pip install django

# 2. 프로젝트 초기화 (현재 폴더에 'config' 폴더 생성)
django-admin startproject config .

# 3. 기능 단위 앱 생성
python manage.py startapp sales
```

---

## 3. 라이브러리 의존성 설정 (`requirements.txt`)
프로젝트 루트 폴더에 `requirements.txt` 파일을 생성하고 영상에서 사용된 핵심 라이브러리들을 추가합니다.

```text
# requirements.txt 파일 생성 후 아래 내용 입력
Django>=4.2, <5.0
djangorestframework>=3.14
psycopg2-binary  # PostgreSQL 사용 시 필수
django-cors-headers
python-dotenv
django-filter
django-extensions
openpyxl         # 엑셀 데이터 처리용
drf-spectacular  # API 문서화(Swagger)
django-silk      # 성능 모니터링용
django-simplejwt # JWT 인증용
```

**설치 명령어:**
```bash
pip install -r requirements.txt
```

---

## 4. 데이터 처리 및 보조 파일 생성
데이터 마이그레이션 및 프로젝트 설명을 위한 빈 파일들을 생성합니다.

```bash
touch readme.md          # 프로젝트 설명
touch excel_data.py      # 엑셀 데이터 처리 로직용
touch model_data.py      # 모델 관련 상수/데이터 관리용
```

---

## 5. 프로젝트 환경 설정 (`config/settings.py`)
생성한 앱과 설치한 라이브러리들을 Django 시스템에 등록합니다.

```python
# config/settings.py 파일 수정

import os
from pathlib import Path

# ... (기본 설정 생략)

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',

    # 외부 라이브러리
    'rest_framework',
    'corsheaders',
    'django_filters',
    'django_extensions',
    'drf_spectacular',

    # 로컬 앱
    'sales',
]

MIDDLEWARE = [
    'corsheaders.middleware.CorsMiddleware',  # 최상단에 배치
    'django.middleware.security.SecurityMiddleware',
    # ... 나머지 기본 미들웨어
]

# DRF 설정 추가
REST_FRAMEWORK = {
    'DEFAULT_SCHEMA_CLASS': 'drf_spectacular.openapi.AutoSchema',
    'DEFAULT_PERMISSION_CLASSES': [
        'rest_framework.permissions.IsAuthenticated',
    ],
    'DEFAULT_AUTHENTICATION_CLASSES': [
        'rest_framework_simplejwt.authentication.JSONWebTokenAuthentication',
    ],
}
```

## 6. 환경 변수 설정 (`.env`)
데이터베이스 정보나 비밀키를 안전하게 관리하기 위해 `.env` 파일을 생성합니다.

```text
# .env 파일 생성 후 입력 (PostgreSQL 예시)
DEBUG=True
SECRET_KEY=django-insecure-your-secret-key
DB_NAME=sales_db
DB_USER=postgres
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=5432
```

---

## 7. 데이터베이스 마이그레이션 및 서버 실행
기본 테이블을 생성하고 서버가 정상적으로 작동하는지 확인합니다.

```bash
# 1. 데이터베이스 변경사항 추적
python manage.py makemigrations

# 2. 데이터베이스 반영
python manage.py migrate

# 3. 서버 실행
python manage.py runserver
```

이제 브라우저에서 `http://127.0.0.1:8000/`에 접속하여 Django 로켓 화면이 보이는지 확인하세요!
